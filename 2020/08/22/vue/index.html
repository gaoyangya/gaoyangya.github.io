发表于：2020-08-22 | 分类： 前端

VUE

即使再小的帆也能远航~

一. 简介

  1. vue是JavaScript框架
  2. 简化Dom操作
  3. 响应式数据驱动

二.VUE入门

1.导入vue开发环境版本,引入 Vue

<!-- 开发环境版本，包含了有帮助的命令行警告 -->
<script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>

2.vue第一个入门应用

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8">
		<title>vue基础</title>
	</head>
	<body>
		<div id="app">
			{{ message }}
			<span>
				<br>
				{{hobby}}
			</span>
			<h3>
				{{user}}  <br />
				姓名:{{user.name}}  <br />
				性别:{{user.sex}}
			</h3>
			<h3>
				{{lists}} <br />
				{{lists[0]}} <br />
				{{lists[1]}} <br />
				{{lists[2]}}
			</h3>
			<h3>
				{{university[0]}}  <br />
				{{university[0].name}} <br />
				{{university[1].local}}
			</h3>
		</div>
		
		<!-- 开发环境版本，包含了有帮助的命令行警告 -->
		<script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>
		
		<script> //这个<script>标签一定要放在有app的<div>标签之后!!!
			var app=new Vue({
				el:"#app",//用来给vue实例定义一个作用范围
				data:{ //用来给vue实例定义一个相关数据
					message:"hello vue!"  ,//相关数据,键值对形式
					hobby:"吃饭睡觉打豆豆~",
					user:{name:"gy",sex:"man"},//定义对象
					lists:["收到","重复","听不见"], //定义数组
					university:[{name:"齐鲁工业大学",local:"济南"},
								{name:"清华大学",local:"北京"}]  //数组中是对象
					
				}
			}); //这个分号";"不写就错了,运行不出来
		</script>
	</body>
</html>

注意:el不仅作用于引入它的标签(eg:<div id="app">),还作用于引入它的标签中的子标签(例如本例中的span)

  • 含有vue实例的<script>标签一定要放在有id=app的<div>标签之后!!!

  • vue实例(对象)中el属性:代表vue的作用范围,日后在vue的作用范围内都可以使用vue语法

  • vue实例(对象)中data属性:用来给vue实例绑定一些相关数据,绑定的数据可以通过{{变量名}}在vue作用范围内取出

  • 在使用{{}}获取data数据时,可以在{{}}中书写表达式,运算符,调用相关方法,以及逻辑运算 eg:{{message.touppercase}}(message内容转大写)

  • el属性中可以书写任意的css选择器,但是在vue开发中推荐使用id选择器

3 v-text和v-html

3.1 区别

  • v-text是用来取值的,把取到的值当成一个文本渲染在页面中
  • v-html是用来取值的,,把取到的值中的html先渲染成html的样式再添加到页面中

3.2 v-text

用来获取data中数据,将数据以文本的形式渲染到指定标签内部,类似于js中的innerText

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8">
		<title></title>
	</head>
	<body>
		<div id="app">
			<span>
			插值表达式:&nbsp;	{{msg}} &nbsp;给我坚持下去~
			</span>  <br/>
			v-test :&nbsp;<span v-text="msg">&nbsp;给我坚持下去~</span>
		</div>
		
		<!-- 开发环境版本，包含了有帮助的命令行警告 -->
		<script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>
		
		<script>
			var app=new Vue({
				el:"#app",
				data:{
					msg:"滴水穿石不是水的力量,而是坚持的力量!"
				}
			});
		</script>		
	</body>
</html>

# 总结
	1.{{}} (插值表达式)和v-text获取数据的区别在于
		(1)使用v-text取值会将标签中原有的数据覆盖,使用差值表达式的形式不会覆盖标签中原有的数据
		(2)使用v-text可以避免在网络环境较差的情况下出现差值闪烁

3.3 v-html

用来获取data中数据,将数据中含有的html标签先解析再渲染到指定标签内部,类似于js中的innerHTML

4vue中事件的定义和使用

4.1 绑定事件语法(v-on)

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8">
		<title></title>
	</head>
	<body>
		<div id="app">
			<h2>age:{{age}}</h2>
			<h2>年龄:{{nianling}}</h2>
			<h2>岁数:{{suishu}}</h2>
			<input type="button" value="点我改变年龄" v-on:click="changeage" />
		</div>
		
		<!-- 开发环境版本，包含了有帮助的命令行警告 -->
		<script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>
		
		<script>
			var app=new Vue({
				el:"#app",
				data:{
					age:29, //int类型
					nianling:"29" ,//字符串类型
					suishu:"29",//字符串类型
				},
				methods:{ //methods 用来定义vue中的事件
					changeage:function(){
					 //在函数中获取vue实例中的data数据
					 //在事件函数中this就是vue实例
					 this.age++;
					 this.nianling++;
					 this.suishu=this.suishu+1;
					}
				}
			});
		</script>		
	</body>
</html>

总结

  • 事件三要素:事件源(发生事件的dom元素);事件(发生特定的行为,如click...);监听器(发生特定动作之后的事件处理程序,通常是js中的函数)

  • 在vue中绑定事件是通过v-on指令来完成的 v-on:事件名 如 v-on:click

  • 在v-on事件名的赋值语句中是当前事件触发调用的函数名 v-on:click="change"

  • 在vue中,事件的函数统一定义在vue实例的methods属性中

  • 在事件函数中this就是vue实例,可以通过this获取vue实例中相关数据

4.2 vue中事件的简化写法(@<==>v-on)

</html>

changeage(){

5 v-show v-if v-blind

5.1 v-show

用来控制页面中某个元素是否展示 底层是display属性

根据表达式的真假,切换元素的显示和隐藏(操纵样式)

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8">
		<title></title>
	</head>
	<body>
		<div id="app">
			<h2 v-show="isShow">我们都被人海裹挟着向前,不能回头!</h2>
			<input type="button" value="切换状态" @click="changeisShow" />
		</div>
		
		<!-- 开发环境版本，包含了有帮助的命令行警告 -->
		<script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>
		
		<script>
			var app=new Vue({
				el:"#app",
				data:{
					isShow:true
				},
				methods:{
					changeisShow:function(){
						this.isShow=!this.isShow;
					}
				}
			});
		</script>		
	</body>
</html>

8 vue中事件修饰符

用来和事件连用,决定事件触发条件或者阻止事件的触发机制

用法:在事件后面加”.xxx” eg: @click=”divClick”

8.1 常用的事件修饰符

修饰符是由点开头的指令后缀来表示的

  • .stop
  • .prevent
  • .capture
  • .self
  • .once
  • .passive
<!-- 阻止单击事件继续传播 -->
<a v-on:click.stop="doThis"></a>

<!-- 提交事件不再重载页面 -->
<form v-on:submit.prevent="onSubmit"></form>

<!-- 修饰符可以串联 -->
<a v-on:click.stop.prevent="doThat"></a>

<!-- 只有修饰符 -->
<form v-on:submit.prevent></form>

<!-- 添加事件监听器时使用事件捕获模式 -->
<!-- 即内部元素触发的事件先在此处理，然后才交由内部元素进行处理 -->
<div v-on:click.capture="doThis">...</div>

<!-- 只当在 event.target 是当前元素自身时触发处理函数 -->
<!-- 即事件不是从内部元素触发的 -->
<div v-on:click.self="doThat">...</div>

8.2 .stop事件修饰符

用来阻止事件冒泡的

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8">
		<title></title>
		<style>
			.aa{
				background-color: red;
				width: 18.75rem;
				height: 18.75rem ;
			}
		</style>
	</head>
	<body>
		<div id="app">
			<div class="aa" @click="divClick">
				<input type="button" value="按钮" @click="btnClick" />
			</div>
		</div>
		
		<!-- 开发环境版本，包含了有帮助的命令行警告 -->
		<script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>
		 
		<script>
			var app=new Vue({
				el:"#app",
				data:{
					
				},
				methods:{
					bthClick(){
						alert("button被点击了")
					},  //没有这个逗号","程序就会报错
					divClick(){
						alert("div被点击了")
					}
				}
			});
		</script>		
	</body>
</html>

点击按钮alert出来的的依旧是"div被点击了"==>事 件冒泡

修改如下:

<div id="app">
	<div class="aa" @click="divClick">
		<input type="button" value="按钮" @click.stop="bthClick" />
	</div>
</div>

		</script>		
	</body>
</html>

点击对话框原先应该是先弹出alert再跳转,现在加上.prevent

8.4 .self事件 修饰符

针对当前标签的事件触发,只触发自己标签上的特定动作事件

只关心自己标签上触发的事件,不监听事件冒泡

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8">
		<title></title>
		<style>
			.aa{
				background-color:red;
				width: 18.75rem;
				height: 18.75rem;
				}
		</style>
	</head>
	<body>
		<div id="app">
			<div class="aa" @click="divClick">
			<input type="button" value="按钮" @click.prevent="btnClick" />
			<input type="button" value="按钮1" @click="btnClick1" />
		</div>
		<!-- 开发环境版本，包含了有帮助的命令行警告 -->
		<script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>
		 
		<script>
			var app=new Vue({
				el:"#app",
				data:{
					
				},
				methods:{
					btnClick(){
						alert("button被点击了")
					},
					btnClick1(){
						alert("button被点击了")
					},
					divClick(){
						alert("div被点击了")
					}
				}
			});
		</script>		
	</body>
</html>

点击这两个按钮,都是先弹出”button被点击了”,然后弹出”div被点击了”

只想点击红色区域的时候才弹出"div被点击了"==>.self

<div id="app">
	<div class="aa" @click.self="divClick">
	<input type="button" value="按钮" @click.prevent="btnClick" />
	<input type="button" value="按钮1" @click="btnClick1" />
</div>

8.5 .once事件修饰符

让指定事件只触发一次

例如上例中的按钮1,点击一次就弹出一次,如果想让其弹出一次

<input type="button" value="按钮1" @click.once="btnClick1" />

9 按键修饰符

  1. 全局组件放在<script>标签中
  2. 全局组件有两个参数
    • 参数1:组件名称,自己取名,在<div>标签(含app那个)中用这个名字去使用全局组件
      • 取名字的时候,如果有多个单词,如果你要是按照驼峰命名法来写,那么在<div id=”app”>里面使用组件名的时候:所有单词小写,用”-“分开

  4. 使用组件的时候在<div id=”app”>里面:<组件名><\组件名>

这里还是说一下安装：npm install vue-router

我们下载vue-cli的时候下载路由了,就不用单独再下载了

13.2 创建组件

如果在一个模块化工程中使用它，必须要通过 Vue.use() 明确地安装路由功能，用vue-cli生产了我们的项目结构，src文件目录下会有一个router文件夹，此处就是编写路由组件的地方。在src/router/index.js，这个文件就是路由的核心文件：

import Vue from 'vue'   //引入Vue
import Router from 'vue-router'  //引入vue-router
import Hello from '@/components/Hello'  //引入根目录下的Hello.vue组件
 
Vue.use(Router)  //Vue全局使用Router
 
export default new Router({
  routes: [              //配置路由，这里是个数组
    {                    //每一个链接都是一个对象
      path: '/',         //链接路径
      name: 'Hello',     //路由名称，
      component: Hello   //对应的组件模板
    }，{
      path:'/hi',
      component:Hi,
      children:[        //子路由,嵌套路由 （此处偷个懒，免得单独再列一点）
        {path:'/',component:Hi},
        {path:'hi1',component:Hi1},
        {path:'hi2',component:Hi2},
      ]
    }
  ]
})

④ yarn概念及安装使用

yarn 是啥

yarn 也是一个软件包管理系统，同样用于管理 用 JavaScript 编写的软件包，yarn 的出现是为了弥补 npm 的一些缺陷。譬如：

1、npm 的问题举例

  • npm install 下载巨慢，特别是新项目，要等半天；删除node_modules，重新install，也一样很慢。
  • 同一个项目，安装无法保持一致性：^5.0.3 表示安装 5.X.X 版本，不同时间部署项目，会 install 不同版本的软件包，给项目带来莫名其妙的问题。

yarn 优化

  • 采用了，并行、离线等功能，优化了 install 时下载速度；
  • 用了 yarn.lock 保证了软件包的版本一致性；
  • 执行命令也比较简洁，如下：
npm install                     === yarn 
npm install taco--save          === yarn add taco
npm uninstall taco--save        === yarn remove taco
npm install taco--save - dev    === yarn add taco--dev
npm update--save                === yarn upgrade

安装yarn

# 安装yarn
npm install yarn -g

# 查看是否安装成功
yarn -v

# 设置yarn源为淘宝镜像源
yarn config set registry https://registry.npm.taobao.org

个人不是很建议安装淘宝镜像,因为会莫名其妙的报一些错

yarn一个启动项目的错

常用

yarn install or npm install

# 启动项目

# 打包构建

⑤ vue-cli的使用

1)vue-cli中项目开发方式

万物皆组件

组件构成:

  • js代码
  • html代码
  • css样式

vue-cli开发方式就是在项目中开发一个一个组件对应一个业务功能模板,日后可以将多个组件组合到一起形成一个前端系统

我们在vue-cli中开发的就是一个个的组件,日后打包时,会将html编译成html文件

主入口是main

//main.js在渲染的时候会被webpack引入变成app.js文件 app.js文件在index.html中会被引入
import Vue from 'vue'	//在页面中引入vue.js
import App from './App'		//引入自定义组件;app自定义是组件名
import router from './router'	//引入vue的router
 
Vue.config.productionTip = false

new Vue({		//vue全局实例
  el: '#app',	//绑定vue实例全局作用范围
  router,		//注册路由对象
  components: { App },	//注册APP组件
  template: '<app />',
  //模板参数必须要有,这里引用了app根组件(根组件对应的html标签形式也就是<app/>),定义的template会在main.js转换为app.js时
  // 以<app/>的形式写在index.html中 而<app/>在最终的渲染结果中并没有体现 可以改写template:'<h1>{{title}}<app/></h1>'测试得出
  data:{
    title:'This is title'
  }
})

日后我们要改就只改app.vue就行了:star::star:
